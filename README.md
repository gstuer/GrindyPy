# GrindyPy

GrindyPy is a Python-based project designed to retrofit **Grind-by-Weight (GbW)** functionality into a coffee grinder.
By integrating a load cell and a microcontroller, this project transforms a standard timed grinder into a precision weight-based dosing system.
This version of grindy is designed to retrofit an **ECM S-Automatik 64** grinder.
A Rust-based partner project for an **Mazzer Mini Electronic B** can be found [here](https://github.com/mkiefel/grindy).

## Hardware Requirements
- Microcontroller capable of running Python (e.g., Raspberry Pi Zero)
- NAU7802 24-Bit ADC
- Load Cell (750g - 1000g)
- 3D-Printed Grinder Parts published [at Onshape](https://cad.onshape.com/documents/78cc3a0852b86e7d625b00cf/w/d49113aa6512174b79b2b171/e/d8b090854583fed821c3b712)
- Nuts & Bolts
